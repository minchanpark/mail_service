## 1. 제품 개요

### 1.1 한 줄 정의

**Inbox One**은 한국 지식근로자가 Gmail·Outlook·Naver Mail을 한 화면에서 관리하고, 모든 메일을 AI가 3줄로 요약·분류해주는 **AI 메일 트리아지 클라이언트**입니다.

### 1.2 왜 지금인가 (Why now)

세 가지 흐름이 동시에 일어나고 있습니다.

1. **다중 메일 계정의 일상화** — 회사 메일·개인 메일·고객 응대 메일·뉴스레터 수신용 메일을 분리해서 쓰는 것이 표준이 되었지만, 정작 통합 클라이언트는 외산(Spark, Edison, Mimestream)뿐이고 모두 한국 메일 서비스(Naver, Daum, Kakao Work) 지원이 빈약합니다.
2. **LLM 기반 요약 품질의 임계점 통과** — Claude·GPT-4 계열 모델은 한국어 비즈니스 메일을 사람 수준으로 요약·분류·답장 초안 생성이 가능합니다. 비용은 메일당 $0.001–0.005로 떨어졌습니다.
3. **Superhuman/Shortwave 검증** — Superhuman은 $30/월에 50만 명, Shortwave는 AI 인박스로 빠르게 성장 중입니다. 이 시장이 존재한다는 것은 검증되었지만, **한국어/한국 메일 환경**에 최적화된 제품은 비어 있습니다.

### 1.3 우리가 해결하는 문제

> "메일함을 여는 데 5분, 다 읽는 데 1시간, 답장에 또 1시간. 그런데 정작 답장이 필요한 건 5통뿐이었다."

타깃 사용자(다음 섹션 참고)의 대표적인 페인 포인트는 다음과 같습니다.

- **계정 전환 피로** — Gmail 탭, Outlook 앱, Naver 웹메일을 번갈아 열어야 함
- **시그널/노이즈 비율 악화** — 뉴스레터·자동 알림·영수증이 진짜 중요한 메일을 묻어버림
- **답장 작성의 정신적 부담** — 짧은 회신도 톤·정중함·맥락 때문에 시작이 어려움
- **메일을 "처리"하는 것이 아니라 "열어보는 것"으로 끝남** — Inbox Zero가 사실상 불가능

## 2. 타깃 사용자

### 2.1 1차 타깃: 한국 스타트업 창업자/팀 리드 (KO-Founder)

- 25–40세, IT/제조/서비스 스타트업 창업자 또는 팀 리드
- 메일 계정 2–4개 (Google Workspace + 개인 Gmail + Naver 또는 Daum)
- 일평균 수신 60–150통, 답장 필요 5–15통
- 영어/한국어 혼용, 가끔 일본어/중국어/스페인어 메일도 받음
- 도구 스택: Slack, Notion, Figma, Linear, GitHub, Toss
- **결제 의향**: 월 $15–25 수준에서 결제 가능 (개인 카드)

### 2.2 2차 타깃: 한국 대기업/외국계 지식근로자 (KO-Knowledge)

- VC, 컨설팅, 법무, 마케팅, PM 등 메일 의존도 높은 직군
- Outlook + 개인 Gmail + Naver 조합이 흔함
- 보안 정책 때문에 Outlook은 회사 계정만 연결 (개인 정보 노출 우려)
- **결제 의향**: 본인은 결제 의향 있으나 회사 결제는 보안 검토 통과 필요 (Phase 2 과제)

### 2.3 페르소나 — "김민지, 33세, 시리즈A 직전 SaaS 창업자"

- 메일: `minji@startup.co.kr` (Google Workspace), `minji.kim@company.com` (이전 직장 Outlook 잔존), `minji_k@naver.com` (개인용)
- 하루: 오전 8시 메일 확인, 점심 후 한 번, 퇴근 전 한 번
- 가장 큰 불만: VC 메일과 고객 CS 메일을 놓치는 일이 1주일에 1–2번 발생
- 이상적인 경험: "출근하자마자 오늘의 메일 브리핑을 보고, 답장 필요한 것만 추려서 30분 안에 끝낸다"

## 3. 핵심 가치 제안

세 가지 약속으로 압축됩니다.

| # | 약속 | 어떻게 | 측정 |
|---|---|---|---|
| 1 | **하나의 인박스** | Gmail/Outlook/Naver를 통합 뷰로 노출, AI 카테고리·라벨로 자동 분류 | 사용자가 다른 메일 앱을 여는 빈도 70% 감소 |
| 2 | **3줄로 끝내는 메일** | 모든 수신 메일에 1줄/3줄 AI 요약 자동 생성, 본문 안 열어도 의사결정 가능 | 메일당 평균 처리 시간 60% 단축 (자체 측정 기준) |
| 3 | **답장은 30초** | AI 답장 초안 3가지 변형 (간결/협조적/정중) 제공, 톤 조절 후 전송 | 답장 1통당 평균 시간 5분 → 1분 |

## 4. 기능 요구사항

### 4.1 MVP 범위 (P0 — 출시 필수)

#### 4.1.1 계정 연결

- **F-101**: Google OAuth로 Gmail 계정 연결 (1개 이상 필수)
- **F-102**: Microsoft OAuth로 Outlook(Microsoft 365 / Outlook.com) 연결
- **F-103**: Naver 메일 연결 — Naver Mail API 가능 시 OAuth, 불가 시 IMAP/SMTP fallback
- **F-104**: 한 사용자당 최대 5개 계정 (가격 등급별 차등은 v1.1)
- **F-105**: 토큰 만료/재인증 안내 UI

#### 4.1.2 통합 인박스

- **F-201**: 모든 연결된 계정의 받은편지함을 시간 역순으로 통합 표시
- **F-202**: 좌측 사이드바에 계정별 필터 (각 계정 미읽음 카운트 포함)
- **F-203**: 상단 탭 필터: 전체 / 읽지 않음 / 답장 필요
- **F-204**: 검색 (`⌘K`) — 발신자, 제목, 본문, 첨부 파일명 across 계정
- **F-205**: 무한 스크롤 (50통 페이지네이션)

#### 4.1.3 AI 요약 (Inbox One의 핵심)

- **F-301**: 각 스레드에 대해 **1줄 요약** 생성 → 인박스 리스트에 표시
- **F-302**: 사용자가 스레드를 열면 **3줄 요약** 표시 (이미 생성됐다면 캐시, 아니면 stream)
- **F-303**: 요약 모델은 Claude (Anthropic API) 기본, 한국어 메일 우선 최적화
- **F-304**: 요약은 비동기 생성 — 메일 수신 시 백그라운드에서 처리, 완료 시 UI 업데이트
- **F-305**: 사용자가 "재요약" 버튼으로 강제 재생성 가능
- **F-306**: 요약 실패 시 명시적으로 표시 (조용히 실패 X)

#### 4.1.4 AI 분류 (카테고리·라벨)

- **F-401**: 모든 메일을 4개 카테고리로 자동 분류 — `중요`, `뉴스레터`, `거래·영수증`, `자동화·알림`
- **F-402**: 사용자가 분류 결과를 수정하면 학습 (피드백 저장, 향후 RAG 또는 fine-tuning 시드)
- **F-403**: 사용자 정의 라벨 (VC·투자, 팀 내부, 고객 CS 등) 생성/수정/삭제
- **F-404**: 라벨은 수동 부여가 기본, 사용자가 라벨에 규칙(rule)을 지정하면 AI가 자동 부여 (v1.1)

#### 4.1.5 AI 답장 초안

- **F-501**: 메일 상세 화면에서 "답장 초안 생성" 버튼 클릭 시 **3가지 톤 변형** 생성
  - 간결 (1–2문장)
  - 협조적 (3–4문장, 일정/조건 명시)
  - 정중·길게 (5–7문장, 비즈니스 격식)
- **F-502**: 사용자가 변형 선택 후 인라인 편집 가능
- **F-503**: 첨부파일 자동 인식 → 답장에 "첨부 검토했습니다" 류 문구 포함 옵션
- **F-504**: 한국어 메일은 한국어 답장, 영어는 영어 (감지 기반)

#### 4.1.6 처리 액션

- **F-601**: 보관 (Archive) — 키보드 `E`
- **F-602**: 삭제 — `⌫`
- **F-603**: 스누즈 — `H` (오늘 저녁 / 내일 아침 / 다음 주 / 사용자 지정)
- **F-604**: 별표 — `S`
- **F-605**: 라벨 부여 — `L`
- **F-606**: 일정 등록 — Google Calendar / Outlook Calendar에 이벤트 생성
- **F-607**: 할일 추가 — 내부 todo 또는 외부 (Notion/Linear) 연동 (v1.1)

#### 4.1.7 오늘의 AI 브리핑

- **F-701**: 매일 사용자가 첫 접속 시 상단에 카드로 표시
  - 수신 / 중요 / 답장 필요 / 자동 분류 통계
  - 오늘 가장 시급한 메일 2–3개
  - 예상 절약 시간 (heuristic 기반)
- **F-702**: 매일 오전 8시(KST) 이메일/푸시로 브리핑 전송 (옵션)

#### 4.1.8 디자인/접근성

- **F-801**: Light / Dark 테마 (시스템 설정 따름 옵션)
- **F-802**: 한국어 폰트 — Pretendard Variable (디자인 시안 기준)
- **F-803**: 키보드 단축키 (Superhuman 패리티 일부): `J/K` 이동, `R` 답장, `C` 새 메일, `⌘K` 검색
- **F-804**: 데스크톱 우선 (1280px+), 모바일 (iOS 18 디자인 기준)은 Phase 2

### 4.2 Phase 2 (P1 — 출시 후 6개월)

- F-901: iOS 네이티브 앱 (또는 Flutter — Park의 강점 영역)
- F-902: 팀 워크스페이스 (라벨/규칙 공유, 메일 위임)
- F-903: AI 자동 답장 규칙 ("뉴스레터 수신 시 자동 보관 + 주간 요약")
- F-904: 첨부파일 AI 분석 (계약서 차이점, 영수증 자동 입력)
- F-905: Slack/Notion/Linear 양방향 연동
- F-906: Daum, Kakao Work 메일 지원

### 4.3 Phase 3 (P2 — 12개월+)

- LATAM 시장 진출 (스페인어 UI/요약, 팀의 스페인어 구사자 자원 활용)

## 5. 비기능 요구사항

### 5.1 성능

- 인박스 첫 로드 (TTFB): **< 1.5s** (한국 사용자 기준, 50통)
- 메일 클릭 → 본문 표시: **< 200ms** (캐시 hit), **< 800ms** (cold)
- AI 요약 생성: **< 3s** (P50), **< 8s** (P95)
- AI 답장 초안: **< 5s** (P50), **< 12s** (P95)
- 동시 접속 1,000 사용자에서 위 지표 유지

### 5.2 보안 & 프라이버시 (Inbox One의 신뢰 기반)

- **OAuth 토큰은 절대 클라이언트에 저장하지 않음** — 서버 측 KMS 암호화 저장
- **메일 본문은 사용자 식별자 없이 LLM에 전달** (요약·답장 생성 시 user_id 전송 X)
- **LLM 학습 데이터로 사용 안 함** — Anthropic API의 zero-retention 옵션 활성화
- **로그**: 메일 본문은 로그에 남기지 않음 (헤더·메타데이터만)
- **데이터 잔류 정책**: 사용자가 계정을 끊으면 7일 내 모든 메일 데이터 영구 삭제
- **암호화**: 전송 중 TLS 1.3, 저장 시 AES-256
- **PIPA/GDPR 준수**: 한국 개인정보보호법 + EU 사용자 대응
- **2FA 권장** (필수는 v1.1)

### 5.3 가용성

- 가용성 목표: **99.5%** (월 ~3시간 다운타임 허용)
- AI 요약은 best-effort: 실패해도 메일 클라이언트 자체는 동작해야 함

### 5.4 국제화 / 현지화

- UI 언어: 한국어 (1차), 영어 (2차)
- 메일 내용 언어 자동 감지 (한/영/일/중/스 5개 언어)

## 6. 사용자 플로우

### 6.1 첫 사용 (온보딩)

```
[랜딩] → [Google 로그인]
     → [Gmail 연결] (필수)
     → [추가 계정 연결] (선택, 스킵 가능)
     → [메일 동기화 진행 화면] (백그라운드, ~30초)
     → [AI 요약 생성 진행] (백그라운드, ~2분 — 사용자는 인박스 사용 가능)
     → [통합 인박스]
```

핵심 원칙: **첫 화면을 보기까지 60초 이내**.

### 6.2 일상 사용 (아침 루틴)

```
[앱 열기]
  → [오늘의 AI 브리핑 카드 자동 표시]
  → [중요 카테고리 클릭 또는 "답장 필요" 탭 클릭]
  → [메일 선택 → 3줄 요약 확인]
  → [답장 초안 생성 → 톤 선택 → 약간 수정 → 전송]
  → [E (보관)]
  → [J (다음 메일)]
  → 반복
```

목표: **이 루틴을 30분 안에 완료**.

### 6.3 답장 플로우 (디테일)

```
[메일 상세] → [답장 초안 생성 클릭]
           → [3개 변형 표시 (간결/협조적/정중)]
           → [변형 선택 → 인라인 편집]
           → [전송 또는 임시저장]
           → [원본 메일은 자동으로 "답장 완료" 상태로 전환]
```

## 7. 성공 지표 (KPI)

### 7.1 활성화 (Activation)

- **A1**: Gmail 연결 완료율 — 가입자의 **80%+**
- **A2**: 첫 메일 처리율 (보관/답장/스누즈) — 가입자의 **60%+**

### 7.2 인게이지먼트 (Engagement)

- **E1**: WAU/MAU — **0.6+** (강한 습관화)
- **E2**: 일평균 세션 수 — **2+ 회**
- **E3**: 일평균 처리 메일 수 — **20+ 통**
- **E4**: AI 답장 초안 사용률 — 발송 답장의 **40%+**

### 7.3 리텐션 (Retention)

- **R1**: D7 retention — **40%+**
- **R2**: D30 retention — **25%+**
- **R3**: 90일 후 유료 전환률 — **15%+**

### 7.4 비즈니스

- **B1**: 6개월 내 유료 사용자 1,000명
- **B2**: ARPU (월) — **$18+**
- **B3**: CAC payback — **6개월 이내**

### 7.5 품질 지표

- **Q1**: AI 요약 정확도 (사용자 "도움이 되었나요?" 긍정률) — **85%+**
- **Q2**: 답장 초안 사용률 (생성된 초안 중 실제 사용된 비율) — **50%+**
- **Q3**: 분류 정확도 (사용자 수동 재분류율) — **10% 이하**

## 11. 리스크 & 가설

### 11.1 핵심 가설 (검증 필요)

1. **H1**: 한국 지식근로자는 다중 메일 계정을 통합 관리할 도구에 월 ₩19,000을 지불한다 → 베타 100명 LOI로 검증
2. **H2**: AI 3줄 요약은 사용자가 본문을 열지 않고 처리할 수 있을 만큼 신뢰도가 높다 → Q1 지표로 측정
3. **H3**: AI 답장 초안의 50%+가 실제 발송에 사용된다 → Q2 지표로 측정

### 11.2 주요 리스크

| 리스크 | 영향 | 완화 |
|---|---|---|
| **Naver Mail API 정책 변경/제한** | 1차 시장 핵심 기능 실패 | IMAP fallback 항상 유지, Daum/Kakao Work 동시 추진 |
| **OAuth 신뢰성 검토** (Google CASA) | Public Beta 지연 가능 | Closed Beta 단계에서 보안 감사 선제적 진행 |
| **LLM 비용 폭증** | 단위경제 붕괴 | 무료 등급에 일 한도, 캐싱 적극 활용, 자체 fine-tuned 소형 모델로 대체 검토 |
| **메일 데이터 유출 사고** | 신뢰 회복 불가, 사실상 회사 종료 | 보안 외주 감사, 버그 바운티, 정기 펜테스트 |
| **Gmail 동기화 지연 (Push API rate limit)** | UX 저하 | Pub/Sub 큐 분산, fallback polling |
| **사용자가 "결국 Gmail 웹을 다시 연다"** | 핵심 가치 실패 | 베타 단계 사용 행동 분석으로 조기 감지 |
